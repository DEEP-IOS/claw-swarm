## 设计理由

LLM 代理没有真实的情感。但它们确实表现出可观察的行为模式，这些模式与情绪状态功能等价：重复失败创造了功能上等同于挫败感的状态；持续成功产生自信；新颖领域触发类似好奇的探索模式。

通过将这些模式建模为连续的情绪维度，Claw-Swarm 能够：

- 在"挫败"代理级联失败之前降低任务复杂度
- 为具有良好记录的"自信"代理启用快速路径路由
- 在"愉悦"水平高时利用积极势头
- 在"紧迫感"上升时优先处理时间敏感任务
- 在"好奇"水平高时鼓励探索
- 在"疲劳"累积时减轻工作负载

情绪智能不是对代理文本内容的情感分析——它是对代理行为模式的结构化量化。

## 6D 情绪向量

每个代理维护一个 6 维情绪向量，通过指数移动平均（EMA）追踪：

| 维度 | 范围 | 信号子类型 | 基线 | 触发条件 |
|------|------|-----------|------|----------|
| **挫败感**（Frustration） | [0, 1] | FRUSTRATION | 0.5 | 重复失败、超时错误、质量门拒绝 |
| **自信**（Confidence） | [0, 1] | CONFIDENCE | 0.5 | 任务成功完成、高质量评分 |
| **愉悦**（Joy） | [0, 1] | JOY | 0.5 | 正面任务结果、用户满意信号 |
| **紧迫感**（Urgency） | [0, 1] | URGENCY | 0.5 | 截止期临近、队列深度、优先级升级 |
| **好奇**（Curiosity） | [0, 1] | CURIOSITY | 0.5 | 新颖任务类型、未探索领域 |
| **疲劳**（Fatigue） | [0, 1] | FATIGUE | 0.5 | 持续高工作负载、长执行链 |

**源代码：** `src/intelligence/social/emotional-state.js`

## EMA 平滑

情绪值通过指数移动平均更新，防止波动性过大：

```
newValue = alpha * observedValue + (1 - alpha) * currentValue
```

核心参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **alpha** | 0.3 | 近期观测权重 30% |
| **基线** | 0.5 | 所有维度从中性点出发 |
| **衰减率** | 0.05/轮 | 未使用的维度向基线漂移 |
| **最大历史** | 200 | 每个代理保留的事件上限 |

这意味着单一事件最多将情绪偏移当前值与观测值差距的 30%。持续的模式会累积复合；孤立的尖峰被抑制。

**衰减机制：** 每轮未被更新的维度自动向基线（0.5）漂移 0.05。这确保临时状态不会无限持续——一个代理如果不再经历失败，其挫败感将自然回归中性。以 0.05/轮的衰减率，一个从 0.9 开始的维度需要约 8 轮才能回到 0.5 附近。

## 路由影响

情绪状态向量反馈到多个决策点：

### 挫败感 > 0.7 → 降低任务复杂度

当挫败感超过 0.7 时：

- `dual-process-router.js` 偏向 System 2（审慎审议）
- 任务 DAG 引擎可能将下一个任务分解为更小的子任务
- 质量门阈值降低，防止拒绝螺旋

**设计原理：** 高挫败代理更可能在复杂任务上失败。降低任务复杂度给予代理恢复信心的机会，打破失败-挫败-再失败的恶性循环。

### 自信 > 0.8 → 快速路径路由

当自信超过 0.8 时：

- `dual-process-router.js` 为熟悉任务启用 System 1（快速路径）
- 对已验证的代理-任务组合减少验证步骤
- 代理有资格接受更高复杂度的分配

**设计原理：** 持续成功的代理已证明其在特定任务类型上的能力。减少不必要的验证开销，同时给予更大挑战，实现渐进式的能力拓展。

### 愉悦 > 0.7 → 积极势头

当愉悦超过 0.7 时：

- 代理获分配更高复杂度的任务以利用积极势头
- 增加协作分配（高愉悦代理提升团队士气）
- 成功连胜奖金放大 Shapley 信用归因

**设计原理：** 积极的情绪状态表明代理处于高效工作状态。将更多资源和挑战分配给高愉悦代理，最大化利用其当前的高效时段。

### 紧迫感 > 0.7 → 优先级升级

当紧迫感超过 0.7 时：

- 任务队列重新排序，优先处理时间敏感项目
- 优先使用 System 1 快速路径路由以最小化延迟
- 可能激活并行执行路径处理关键任务

**设计原理：** 高紧迫感信号意味着外部截止期限或资源竞争正在升级。系统需要自动将决策速度提升到与紧迫程度匹配的水平。

### 好奇 > 0.6 → 探索许可

当好奇超过 0.6 时：

- ABC 调度器可能将该代理分配为侦察蜂（5% 探索角色）
- 新颖任务类型优先分配，再回退到常规分配
- 学习率（EMA alpha）可能临时提高

**设计原理：** 好奇状态表明代理对新领域有学习潜力。利用这一时机进行探索，可在不牺牲整体效率的情况下拓展蜂群的知识边界。

### 疲劳 > 0.7 → 减轻工作负载

当疲劳超过 0.7 时：

- 降低该代理的最大并发子代理数
- 降低任务复杂度上限
- 在任务分配之间安排恢复期

**设计原理：** 持续高负载会降低代理输出质量。及时减轻工作量，让代理有机会恢复到最佳状态，比强制继续导致质量下降后再修复成本更低。

## 信号集成

情绪状态通过信号-网格（Signal-Mesh）以 EMOTIONAL 信号类型传播：

```javascript
field.deposit(createSignal({
  type: SignalType.EMOTIONAL,
  subtype: SignalSubtype.FRUSTRATION,
  source: { kind: SourceKind.WORKER, id: agentId },
  origin: `/agent/${agentId}`,
  intensity: 0.8,
  payload: {
    agentId,
    dimension: 'frustration',
    value: 0.82,
    trigger: 'quality_gate_rejection'
  }
}))
```

任何拥有 EMOTIONAL 受体的 ModuleBase 都可以对情绪信号做出反应——实现蜂群级的"共情"，各引擎基于集体情绪状态调整行为。

**传播特性：** EMOTIONAL 信号通过 12 维信号场传播。单个代理的挫败信号会被其他已注册 EMOTIONAL receptor 的模块感知，当多个代理同时表现出高挫败时，触发系统级的降负载响应。

## 文化摩擦

**源代码：** `src/intelligence/social/cultural-friction.js`

当使用不同 LLM 模型的代理协作时，"文化"差异产生摩擦：

| 摩擦维度 | 描述 | 示例 |
|----------|------|------|
| 输出格式不匹配 | 结构化 vs 叙述式 | 一个模型输出 JSON，另一个输出散文 |
| 推理风格差异 | 逐步 vs 整体 | 一个模型展示推导过程，另一个直接给结论 |
| 沟通冗余度 | 简洁 vs 详细 | 响应长度差异 3 倍以上 |
| 错误处理方式 | 显式 vs 隐式 | 一个模型明确报错，另一个静默继续 |

文化摩擦模型量化这些成本，并反馈到任务分配流程。当检测到两种模型类型之间的高摩擦时，系统优先将同"文化"代理分配到紧密耦合的任务，或在跨文化协作中插入翻译层。

## 意义建构引擎

**源代码：** `src/intelligence/social/self-reflection.js`

当复杂的多代理任务产生意外结果时，意义建构引擎建构连贯叙事：

1. **回顾性分析** — 回顾信号、决策和结果的时序序列
2. **模式提取** — 识别跨代理的重复情绪模式
3. **叙事建构** — 构建关于发生了什么以及为什么的解释
4. **学习整合** — 将洞察反馈到情节记忆供未来参考

意义建构不追求"准确性"——它追求"合理性"。在复杂涌现系统中，完全准确的因果归因往往不可能。但一个合理的叙事足以指导未来决策和策略调整。

## 偏见检测

**源代码：** `src/quality/analysis/anomaly-detector.js`

认知偏见影响代理行为，正如它们影响人类决策。偏见检测器监控以下四类偏见：

| 偏见类型 | 描述 | 检测方法 |
|----------|------|----------|
| **锚定偏见** | 过度依赖初始信息 | 监控决策对首个输入的敏感度 |
| **确认偏见** | 寻找确认现有方法的证据 | 检测信息搜索的偏向性 |
| **可得性偏见** | 过度加权近期或突出的例子 | 比较决策中近期 vs 远期事件的影响力 |
| **沉没成本谬误** | 因先前投入而坚持失败策略 | 监控任务切换阻力与累积投入的相关性 |

偏见检测器监控代理决策模式并标记潜在偏见，使编排层能在偏见决策级联之前进行干预。

## 配置

情绪智能不需要额外配置。所有参数有仿生启发的默认值：

```json
{
  "emotionalIntelligence": {
    "emaAlpha": 0.3,
    "baseline": 0.5,
    "decayRate": 0.05,
    "maxHistory": 200
  }
}
```

| 参数 | 默认值 | 范围 | 调优建议 |
|------|--------|------|----------|
| `emaAlpha` | 0.3 | (0, 1) | 增大使情绪更敏感，减小使情绪更稳定 |
| `baseline` | 0.5 | [0, 1] | 通常不需要修改 |
| `decayRate` | 0.05 | (0, 0.5) | 增大使情绪更快回归中性 |
| `maxHistory` | 200 | [50, 1000] | 增大保留更多历史，占用更多内存 |

所有配置位于 `~/.openclaw/openclaw.json` → `plugins.entries.openclaw-swarm.config.emotionalIntelligence`。
